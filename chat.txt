## FILE PERMISSIONS (chmod) - COMPLETE BREAKDOWN

### The Permission System

Linux has 3 user categories for every file:

Category | Letter | Meaning
---|---|---
Owner | u | The user who created the file
Group | g | Users in the same group as the file
Others | o | Everyone else on the system

And 3 permission types:

Permission | Letter | Number | What It Allows
---|---|---|---
Read | r | 4 | View file contents / List directory
Write | w | 2 | Modify file / Add or remove files in directory
Execute | x | 1 | Run file as program / Enter directory

### How Numbers Work (Octal Notation)

Each digit = Owner + Group + Others

Number | Binary | Permission | Meaning
---|---|---|---
0 | 000 | --- | No permissions
1 | 001 | --x | Execute only
2 | 010 | -w- | Write only
3 | 011 | -wx | Write + Execute
4 | 100 | r-- | Read only
5 | 101 | r-x | Read + Execute
6 | 110 | rw- | Read + Write
7 | 111 | rwx | Full permissions

### Common `chmod` Values Explained

Command | Number Breakdown | Result | Use Case
---|---|---|---
`chmod 777 file` | 7(rwx) 7(rwx) 7(rwx) | Everyone can do everything | DANGEROUS - never use in production
`chmod 755 file` | 7(rwx) 5(r-x) 5(r-x) | Owner full, others read+execute | Scripts, programs, directories
`chmod 644 file` | 6(rw-) 4(r--) 4(r--) | Owner read+write, others read only | Config files, documents
`chmod 600 file` | 6(rw-) 0(---) 0(---) | Only owner can read+write | Secrets, passwords, private keys
`chmod 666 file` | 6(rw-) 6(rw-) 6(rw-) | Everyone read+write | Shared log files (rarely used)

Why 777 is dangerous: Any user can modify your file. If it's a script, they can inject malicious code.

### Your Specific Files - What Should They Be?

File | Permission | Why
---|---|---
`main.py` | 755 | Needs to be executable (run as program), but others shouldn't modify it
`config.txt` | 644 | Others need to read config, but only owner should change settings
`app.log` | 666 or 644 | Log files often need to be writable by multiple processes
`backup.tar` | 600 | Backups contain sensitive data - owner only

## USER MANAGEMENT COMMANDS - WORD BY WORD

### 1. `whoami`
- **What it does**: Prints the username you're currently logged in as
- **Why use it**: To confirm which user account is active
- **Example output**: `devops`
- **Real use**: When you switch users (`su`), you forget who you are. This reminds you.

### 2. `id`
- **What it does**: Shows your user ID (UID), group ID (GID), and all groups you belong to
- **Why use it**: To see your permissions across the system
- **Example output**:
  ```
  uid=1000(devops) gid=1000(devops) groups=1000(devops),4(adm),24(cdrom),27(sudo)
  ```
- **Breakdown**:
  - `uid=1000` - Your unique user number (1000+ for regular users, 0 is root)
  - `gid=1000` - Your primary group's number
  - `groups=` - All groups you belong to (`sudo` group means you can use `sudo`)

### 3. `sudo adduser devopsuser --gecos "" --disabled-password`

Word by word:

Part | Meaning
---|---
`sudo` | "Superuser Do" - run this command as root (administrator)
